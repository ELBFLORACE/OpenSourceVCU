# Contributing

Thank you for considering contributing to this project. Contributions of all kinds are welcome and appreciated.

## How to Contribute

There are two primary ways to contribute:

### Reporting Issues

* Use the issue tracker to report bugs, request features, or suggest improvements.
* Before opening a new issue, please check if a similar one already exists.
* Provide clear and concise information, including:

  * A description of the problem or idea
  * Steps to reproduce (for bugs)
  * Expected and actual behavior

### Submitting Pull Requests

* Fork the repository and create a dedicated branch for your changes.
* Keep pull requests focused and limited in scope.
* Provide a clear description of your changes and the motivation behind them.
* Reference related issues where applicable.

## Code Quality Standards

To maintain consistency and quality across the codebase:

* **Autoformatting is required**
  All submitted code must be formatted using the project's agreed formatting tools before submission.

* **Tests are mandatory for new features**
  Any new feature must include tests that cover the majority of the added functionality.

* **Maintain readability and consistency**
  Follow the existing code style and structure. Write clear, maintainable code.

* **Keep changes minimal and focused**
  Avoid mixing unrelated changes in a single pull request.

## Review Process

* All contributions are reviewed before being merged.
* Feedback may be provided—please be open to revisions and discussion.
* Collaborative improvement is encouraged.

## Final Notes

Contributions help improve the project for everyone. Whether it's fixing a bug, improving documentation, or proposing a new idea, your effort is valued and appreciated.
